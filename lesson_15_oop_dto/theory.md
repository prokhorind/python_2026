## ООП та Data Transfer Objects (DTO) у Python

### 🔹 Що таке ООП?

Об'єктно-орієнтоване програмування (ООП) — це парадигма програмування, яка базується на використанні об'єктів.

Основні принципи:

* **Інкапсуляція** — об'єднання даних і методів
* **Наслідування** — створення нових класів на основі існуючих
* **Поліморфізм** — однаковий інтерфейс для різних типів
* **Абстракція** — приховування складності

---

### 🔹 Що таке DTO?

**DTO (Data Transfer Object)** — це простий об'єкт, який використовується для передачі даних між частинами програми.

👉 DTO **НЕ містить бізнес-логіки**
👉 DTO містить **лише дані**

### 🔹 Навіщо використовувати DTO?

* Відокремлення логіки від даних
* Зручна передача даних (API, БД)
* Спрощення тестування
* Чиста архітектура

---

### 🔹 Приклад DTO

```python
class UserDTO:
    def __init__(self, username, email):
        self.username = username
        self.email = email
```

👉 Тут немає логіки — тільки дані

---

### 🔹 DTO vs Звичайний клас

| DTO                           | Звичайний клас                 |
| ----------------------------- | ------------------------------ |
| Тільки дані                   | Дані + логіка                  |
| Простий                       | Складніший                     |
| Використовується для передачі | Використовується для поведінки |

---

### 🔹 Де використовується DTO?

* API (запити/відповіді)
* Робота з базами даних
* Передача даних між шарами (UI → Logic → DB)

---

### 🔹 Важлива ідея

👉 DTO = "контейнер для даних"
👉 Ніяких складних методів
👉 Максимально проста структура
