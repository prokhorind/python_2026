# Linked List (Звʼязаний список) — теорія та приклади (Python)

## 1. Що таке Linked List
**Linked List** — це структура даних, де кожен елемент (вузол, node)
зберігає:
- значення
- посилання на наступний елемент

Елементи **не зберігаються поруч у памʼяті**.

```text
[value | next] -> [value | next] -> [value | None]
```

---

## 2. Порівняння з масивом

| Властивість | Array | Linked List |
|-----------|------|-------------|
| Доступ за індексом | O(1) | O(n) |
| Вставка на початок | O(n) | O(1) |
| Вставка в середину | O(n) | O(1)* |
| Памʼять | Менше | Більше |

\* якщо ми вже знаємо потрібний вузол

---

## 3. Структура вузла (Node)

Кожен вузол зберігає:
- `value` — значення
- `next` — посилання на наступний вузол

```python
class Node:
    def __init__(self, value):
        self.value = value
        self.next = None
```

---

## 4. Основні операції

### Додавання на початок
```text
new_node -> old_head -> ...
```

- O(1)
- не потребує зсуву елементів

---

### Додавання в кінець
- потрібно пройти весь список
- O(n)

---

### Пошук елемента
- послідовний перегляд
- O(n)

---

## 5. Чому Linked List немає "з коробки" в Python

> Python оптимізований під масиви та кеш-памʼять процесора.
> У реальних задачах `list` майже завжди швидший.

Але:
- Linked List — **ідеальний навчальний приклад**
- допомагає зрозуміти посилання та памʼять

---

## 6. Коли Linked List має сенс
- часті вставки / видалення
- відсутня потреба у доступі за індексом
- реалізація інших структур (stack, queue)

---

## 7. Висновок
> Linked List — це структура, де швидкість вставки
> досягається ціною повільного доступу.

Вона протилежна масиву за своїми властивостями.
