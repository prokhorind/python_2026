# Arrays (Масиви) — теорія та приклади (Python)

## 1. Що таке масив
**Масив** — це структура даних, яка зберігає елементи **поспіль у памʼяті** та дозволяє
швидкий доступ до елемента за індексом.

У класичних мовах (C, Java):
- масив **однотипний**
- кожен елемент займає однакову кількість памʼяті

У Python:
- `list` — це **динамічний масив**
- зберігає **посилання на обʼєкти**, а не самі значення

---

## 2. Однотипність масивів

### Класичний масив (ідея)
```c
int arr[3] = {1, 2, 3};
```

### Python
```python
a = [1, "hello", 3.14]
```

✅ Python дозволяє різні типи, бо фактично зберігає:
```text
[ ref ][ ref ][ ref ]
```

Формулювання для учнів:
> Масив у Python — це масив **посилань**, а не значень.

---

## 3. Статичний vs динамічний масив

### Статичний
- фіксований розмір
- не може розширюватись

```c
int arr[5];
```

### Динамічний (Python `list`)
```python
a = []
a.append(10)
a.append(20)
```

Під капотом:
- створюється новий масив більшого розміру
- старі елементи копіюються

---

## 4. Що реально зберігається в list

```python
a = [1, 2, 3]
b = a
```

```text
a ─┐
   ├──> [ ref ][ ref ][ ref ]
b ─┘
```

Зміни через `b` впливають на `a`.

---

## 5. Доступ за індексом — O(1)

У класичних мовах:
```text
address = base_address + index * element_size
```

У Python:
- `element_size` = розмір посилання
- всі посилання однакового розміру

Тому:
```python
a[5]
```
працює за **O(1)**

---

## 6. Вставка та видалення

```python
a.insert(0, 100)
```

- всі елементи зсуваються
- складність: **O(n)**

---

## 7. Переваги та недоліки масивів

### Плюси
- швидкий доступ
- компактне зберігання

### Мінуси
- повільні вставки всередині
- копіювання при розширенні

---

## 8. Висновок
> Масив — це швидкість доступу за індексом ціною повільних вставок.

Python будує більшість контейнерів саме на цій ідеї.
