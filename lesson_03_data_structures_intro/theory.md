## Теоретичний матеріал

### 1. Що таке структури даних?

Структури даних - це способи організації та зберігання даних у програмі для ефективного доступу та модифікації.

```python
# Прості типи даних (одне значення)
name = "Олексій"
age = 16

# Структури даних (багато значень)
grades = [10, 8, 9, 7]  # Список
student = {"name": "Олексій", "age": 16}  # Словник
```

### 2. Списки (Lists)

Списки зберігають упорядковану колекцію елементів.

```python
# Створення списків
fruits = ["яблуко", "банан", "апельсин"]
numbers = [1, 2, 3, 4, 5]
mixed = ["текст", 42, True, 3.14]

# Доступ до елементів
print(fruits[0])    # "яблуко" (перший елемент)
print(fruits[-1])   # "апельсин" (останній елемент)

# Зміна елементів
fruits[1] = "груша"
print(fruits)  # ["яблуко", "груша", "апельсин"]

# Довжина списку
print(len(fruits))  # 3
```

### 3. Словники (Dictionaries)

Словники зберігають пари "ключ-значення".

```python
# Створення словника
student = {
    "name": "Марія",
    "age": 17,
    "grade": 11,
    "subjects": ["математика", "фізика", "інформатика"]
}

# Доступ до значень
print(student["name"])     # "Марія"
print(student["age"])      # 17

# Додавання нових пар
student["city"] = "Київ"

# Зміна значень
student["age"] = 18

print(student)
```

### 4. Кортежі (Tuples)

Кортежі - це незмінні списки.

```python
# Створення кортежу
coordinates = (10, 20)
rgb_color = (255, 128, 0)
student_info = ("Іван", 16, "10-А")

# Доступ до елементів
x, y = coordinates  # Розпакування
print(f"X: {x}, Y: {y}")

# Кортежі не можна змінювати
# coordinates[0] = 15  # Помилка!

# Але можна створити новий
new_coordinates = (15, 25)
```

### 5. Множини (Sets)

Множини зберігають унікальні елементи.

```python
# Створення множини
favorite_colors = {"червоний", "синій", "зелений"}
numbers = {1, 2, 3, 3, 4, 4, 5}  # Дублікати видаляються
print(numbers)  # {1, 2, 3, 4, 5}

# Додавання елементів
favorite_colors.add("жовтий")

# Перевірка наявності
if "червоний" in favorite_colors:
    print("Червоний є у списку!")

# Операції з множинами
set1 = {1, 2, 3, 4}
set2 = {3, 4, 5, 6}

print(set1 & set2)  # Перетин: {3, 4}
print(set1 | set2)  # Об'єднання: {1, 2, 3, 4, 5, 6}
```

### 6. Вибір правильної структури даних

```python
# Список - коли потрібен порядок та можливість змін
shopping_list = ["хліб", "молоко", "яйця"]

# Словник - коли потрібен швидкий доступ за ключем
phone_book = {"Мама": "+380501234567", "Тато": "+380507654321"}

# Кортеж - для незмінних даних
birth_date = (15, 8, 2007)  # день, місяць, рік

# Множина - для унікальних елементів
visited_cities = {"Київ", "Львів", "Одеса"}
```

## Ключові поняття
- **Список** - упорядкована змінна колекція
- **Словник** - колекція пар "ключ-значення"
- **Кортеж** - упорядкована незмінна колекція
- **Множина** - неупорядкована колекція унікальних елементів
- **Індекс** - позиція елемента у списку або кортежі
- **Ключ** - унікальний ідентифікатор у словнику
