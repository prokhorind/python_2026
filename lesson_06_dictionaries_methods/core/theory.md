# Структура даних Dictionary у Python

## 1. Що таке dictionary?

Dictionary — це структура даних типу:

КЛЮЧ → ЗНАЧЕННЯ

Приклад:

{
    "name": "Anna",
    "age": 16
}

---

# 2. На якій структурі даних побудований dictionary?

Dictionary у Python побудований на основі:

ХЕШ-ТАБЛИЦІ (Hash Table)

А хеш-таблиця — це:

динамічний масив + функція хешування

---

# 3. Як працює хешування?

## Крок 1. Обчислення хешу

Python викликає:

hash(key)

Наприклад:

hash("cat") → 583920134

---

## Крок 2. Обчислення індексу

index = hash(key) % size

Припустимо:
size = 10

583920134 % 10 = 4

Отже елемент буде збережено в комірці №4.

---

# 4. Візуалізація вставки

Нехай таблиця має розмір 7:

Індекс:

0 | 1 | 2 | 3 | 4 | 5 | 6
--|---|---|---|---|---|---
  |   |   |   |   |   |

Вставляємо:

dict["cat"] = 5

1️⃣ hash("cat") = 583920134  
2️⃣ 583920134 % 7 = 3  
3️⃣ Записуємо в комірку 3

0 | 1 | 2 | 3           | 4 | 5 | 6
--|---|---|-------------|---|---|---
  |   |   | ("cat", 5)  |   |   |

---

# 5. Колізії (collision)

Якщо:

hash("dog") % 7 = 3

Але комірка 3 вже зайнята!

Тоді Python використовує open addressing:

Шукаємо наступну вільну комірку:

3 → 4 → 5 → ...

---

# 6. Алгоритм вставки (спрощено)

1. Обчислити hash(key)
2. Обчислити index = hash % size
3. Якщо комірка порожня → вставити
4. Якщо ключ вже існує → оновити значення
5. Якщо зайнята іншим ключем → перейти до наступної комірки
6. Повторювати поки не знайдеться місце

---

# 7. Візуалізація пошуку

Шукаємо dict["cat"]

1️⃣ hash("cat")  
2️⃣ index = hash % size  
3️⃣ Перевіряємо комірку  
4️⃣ Якщо ключ співпадає → повертаємо значення  
5️⃣ Якщо ні → рухаємось далі  

---

# 8. Алгоритм пошуку

1. Обчислити hash(key)
2. Обчислити index
3. Якщо комірка порожня → ключа немає
4. Якщо ключ співпадає → повернути значення
5. Інакше → перейти до наступної комірки
6. Повторювати поки:
   - знайдемо ключ
   - або зустрінемо порожню комірку

---

# 9. Що реально зберігається всередині?

Кожен запис містить:

- hash ключа
- ключ
- значення

Тобто елемент виглядає як:

(hash, key, value)

---

# 10. Складність

Середній випадок:

Вставка — O(1)  
Пошук — O(1)  
Видалення — O(1)

Гірший випадок (багато колізій):

O(n)

---

# 11. Чому dictionary дуже швидкий?

Бо:

- Не потрібно проходити всю структуру
- Доступ відбувається через обчислення індексу
- Це не пошук — це пряме звернення до комірки
