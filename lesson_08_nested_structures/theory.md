# Graph та Binary Search Tree у Python

## Вкладені структури даних

У програмуванні багато складних структур даних будуються з простіших.
Такі структури називаються **вкладеними (composite data structures)**.

Приклади:

* `list` → динамічний масив
* `dict` → hash table
* `graph` → dictionary + lists
* `tree` → nodes + references

У цьому уроці розглянемо дві важливі структури:

* **Graph (граф)** — використовується для представлення мереж
* **Binary Search Tree (BST)** — використовується для швидкого пошуку даних

---

# Graph

## Що таке граф

**Graph** — це структура даних, яка складається з:

* **вершин (nodes / vertices)**
* **ребер (edges)** між вершинами

Графи використовуються для опису **зв'язків між об'єктами**.

Приклади реальних систем:

* соціальні мережі
* карти доріг
* інтернет
* мережі комп'ютерів
* логістичні маршрути

---

# Приклад графу

Уявімо просту соціальну мережу:

* **A знає B і C**
* **B знає A і D**
* **C знає A**
* **D знає B**

Візуально це виглядає так:

```
    A
   / \
  B   C
  |
  D
```

Зв'язки:

* A ↔ B
* A ↔ C
* B ↔ D

---

# Як граф зберігається у Python

Найпоширеніше представлення — **adjacency list (список суміжності)**.

Це комбінація:

```
dictionary + lists
```

Приклад у Python:

```python
graph = {
    "A": ["B", "C"],
    "B": ["A", "D"],
    "C": ["A"],
    "D": ["B"]
}
```

Тут:

* ключ словника — **вершина**
* список — **всі вершини, з якими вона з'єднана**

Наприклад:

```
A → B, C
```

означає, що A має зв'язок з B і C.

---

# Обхід графу

Щоб дослідити граф, використовують **алгоритми обходу**.

---

## Depth First Search (DFS)

Алгоритм рухається **якнайглибше у графі**.

Використовує:

* **stack**
* або **рекурсію**

Приклад обходу:

```
A → B → D → C
```

---

## Breadth First Search (BFS)

Алгоритм рухається **по рівнях графу**.

Використовує:

* **queue**

Приклад обходу:

```
A → B → C → D
```

---

# Чи є дерево графом?

Так. **Дерево — це спеціальний вид графу.**

Граф — це загальна структура з вершинами та ребрами.

Дерево — це граф, який має додаткові правила.

---

# Властивості дерева

Дерево — це граф, який:

1. **не має циклів**
2. має **один корінь (root)**
3. між будь-якими двома вузлами існує **тільки один шлях**

Приклад дерева:

```
        A
       / \
      B   C
     / \
    D   E
```

---

# Приклад графу з циклом

```
A --- B
|     |
C --- D
```

У цьому графі є цикл:

```
A → B → D → C → A
```

Тому це **граф**, але **не дерево**.

---

# Основні відмінності Graph та Tree

| Властивість      | Graph            | Tree        |
| ---------------- | ---------------- | ----------- |
| Цикли            | можуть бути      | немає       |
| Корінь           | немає            | є           |
| Шлях між вузлами | може бути багато | тільки один |
| Структура        | мережа           | ієрархія    |

Важливо:

```
Tree ⊂ Graph
```

Тобто **кожне дерево є графом, але не кожен граф є деревом**.

---

# Binary Tree

**Бінарне дерево** — це дерево, де кожен вузол має максимум **2 дітей**:

* `left`
* `right`

Приклад:

```
      A
     / \
    B   C
```

---

# Binary Search Tree (BST)

**Binary Search Tree** — це спеціальний тип бінарного дерева.

Правило BST:

```
left < node < right
```

Приклад:

```
        8
       / \
      3   10
     / \    \
    1   6    14
```

---

# Властивість BST

Для кожного вузла:

* всі значення **зліва менші**
* всі значення **справа більші**

Це дозволяє дуже швидко знаходити елементи.

Середня складність операцій:

```
O(log n)
```

---

# Як виглядає вузол дерева у Python

Кожен вузол містить:

* `value`
* `left`
* `right`

Приклад:

```python
class Node:

    def __init__(self, value):
        self.value = value
        self.left = None
        self.right = None
```

---

# Обхід Binary Search Tree

Є три основні способи обходу дерева.

---

## Inorder

```
Left → Node → Right
```

Для BST цей обхід повертає **відсортовані дані**.

---

## Preorder

```
Node → Left → Right
```

---

## Postorder

```
Left → Right → Node
```

# Практичне використання Graph

Graph використовується у:

* соціальних мережах
* GPS навігації
* аналізі мереж
* пошуку найкоротшого маршруту
* рекомендаційних системах

---

# Практичне використання BST

Binary Search Tree використовується у:

* базах даних
* файлових системах
* пошукових індексах
* компіляторах
* системах автодоповнення

---

# Висновок

Graph і Binary Search Tree — фундаментальні структури даних.

Graph дозволяє описувати **мережі та зв'язки між об'єктами**.

Binary Search Tree дозволяє **ефективно зберігати та швидко шукати дані**.
